## 04 项目描述

**核心痛点：**

数学学习过程中，讲义通常是线性的"章→节→知识点"结构，但人脑的记忆和理解是网状关联的。手动将讲义转化为 Obsidian 双向链接知识库极其耗时，跨文档的语义关联很难靠人工发现。传统笔记工具缺乏对 LaTeX 公式、数学推导等专业内容的支持，且多门课程之间的知识点无法自动建立连接，形成"信息孤岛"。

**核心逻辑流（基于 LangGraph 的长链推理流水线）：**

1. **多格式输入** — 支持 `.md` / `.tex` / `.pdf` 三种格式，PDF 经 LlamaParse 转换为 Markdown
2. **宏观知识图谱分析** — LLM 通读全文，提取章节→知识点的树状结构（消耗约 4K tokens/篇）
3. **向量库入库** — ChromaDB 切片（1000字符/片，重叠150字符）并向量化存储，自动去重
4. **全局索引生成** — 自动生成 `_00_Index_文档名.md` MOC 索引笔记
5. **并行知识点处理** — ThreadPoolExecutor 多线程（默认4线程，最高8线程），每个知识点独立走以下流水线：
   - ① ChromaDB 语义检索 — 从全库召回相关上下文，优先同文档过滤防跨学科污染
   - ② LLM 生成笔记 — System prompt 严格约束输出格式，含 Prompt 泄漏防护
   - ③ LaTeX 闭合校验 — 检查 `$$` 和 `$` 成对闭合
   - ④ 逻辑审计 — 审计模型交叉验证推导准确性，不通过则带反馈重试（最多2次）
   - ⑤ 保存 + 语义链接 — 写入 Obsidian 并追加全库语义相似度最高的 Top-3 笔记链接

**技术亮点：**

- **长链推理闭环**：宏观分析→微观生成→质量审计→反馈重试，形成推理-验证的完整闭环，每条笔记平均经过 2-3 轮 LLM 调用
- **质量保障机制**：LaTeX 公式成对校验、审计模型交叉验证、Prompt 泄漏自动清洗（移除代码块包裹、角色扮演文本、元文本）、最多 2 次重试
- **跨学科污染防护**：ChromaDB 检索优先按文档名精确匹配，逐级降级：精确匹配 → 补 `.md` → 不限来源
- **并发加速**：ThreadPoolExecutor 4-8 线程并行处理知识点，同时可运行文件监听模式（watchdog）实现拖入即处理
- **TUI 实时监控**：基于 Rich 的仪表盘显示进度、成功率、错误分布

## 05 使用证明

### ✅ 已就绪

- **GitHub 仓库**：https://github.com/Yiwan233/lecture2obsidian
  - 完整的 README、流程图、项目文档
  - 清晰的提交历史和项目结构

### 📸 建议补充的截图材料

| 材料 | 获取方式 |
|------|----------|
| **终端运行日志** | 执行 `python watcher.py -a -c 4` 截图终端输出 |
| **TUI仪表盘** | 执行 `python tui.py data/*.md` 截图 Rich 面板 |
| **Obsidian图谱** | 处理后在 Obsidian 中打开 Graph View 截图 |
| **DeepSeek账单** | platform.deepseek.com → 用量 → 近30天截图 |
| **生成笔记示例** | 打开 Obsidian 库中任意生成的 `.md` 笔记截图 |

### 运行截图生成脚本

```bash
python watcher.py -a -c 4
```

处理完成后，打开 Obsidian 图谱视图截图即可。
